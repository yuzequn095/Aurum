# Aurum AI Architecture

## Overview

Aurum is designed as an **AI-driven financial intelligence platform**.

AI in Aurum is responsible for transforming raw financial data into actionable insights, financial guidance, and decision support.

The AI system operates across three layers:

1. Insight Engine
2. AI Conversation System
3. Financial Intelligence Models

```text
Financial Data
  |
  v
Analytics Layer
  |
  v
Insight Engine
  |
  v
AI Layer
  |
  v
User Insights & Conversations
```

---

## Insight Engine

The Insight Engine is responsible for generating financial insights from structured financial data.

It operates using a **hybrid architecture**.

### Insight Engine Modes

The system supports three modes:

`rules`
`llm`
`hybrid`

#### Rules Mode

Uses deterministic logic.

Examples:

- spending spike detection
- category dominance
- month-over-month changes
- income/expense imbalance

Advantages:

- deterministic
- fast
- zero AI cost

Used as the **baseline insight system**.

---

#### LLM Mode

Insights are generated by an LLM.

Example prompts:

- financial summaries
- behavioral spending analysis
- financial advice

Advantages:

- more contextual analysis
- natural language explanations

Disadvantages:

- API cost
- slower

---

#### Hybrid Mode

Hybrid mode combines rules and LLM.

Process:

1. rules engine generates structured insights
2. LLM refines and summarizes insights

Example pipeline:

```text
Financial Data
  |
  v
Rules Engine
  |
  v
Structured Insights
  |
  v
LLM Refinement
  |
  v
Natural Language Report
```

Hybrid mode provides the best balance between cost and intelligence.

## AI Conversation System

Aurum provides two types of AI conversations.

### Quick AI Chat

Temporary AI interaction.

Characteristics:

- ephemeral session
- not saved by default
- optional save

Example user queries:

- "How much did I spend this month?"
- "What was my largest expense last year?"

If the user saves the conversation, it moves into the persistent conversation system.

---

### Persistent AI Conversations

Stored inside the AI Insights module.

Examples:

- Monthly Report
- Quarterly Review
- Yearly Report
- Budget Advisor
- Portfolio Analysis

These conversations behave like **AI financial advisors**.

They maintain conversation history and context.

---

## Prompt Architecture

Aurum uses structured prompts.

Prompts are organized into a **Prompt Pack**.

Example prompt categories:

`monthly_report_prompt`
`yearly_summary_prompt`
`budget_analysis_prompt`
`portfolio_analysis_prompt`

Prompts will be stored in the repository.

Example structure:

```text
prompts/
  monthly-report.md
  yearly-report.md
  budget-analysis.md
```

Each prompt defines:

- system instructions
- data format
- expected output structure

This ensures AI responses remain consistent.

## AI Cost Strategy

To control operational costs, Aurum follows a layered strategy.

Rules Engine -> default insights

LLM -> optional advanced analysis

Hybrid Mode -> balance between cost and intelligence

This approach allows Aurum to provide AI functionality without excessive API usage.
